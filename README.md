# گزارش پروژه: انیمیشن کاراکتر سفارشی با دست‌های بالا و پایین روی LCD 

## ۱. مقدمه  
در این پروژه، دو کاراکتر سفارشی به نام `armsDown` و `armsUp` طراحی شده‌اند که نشان‌دهنده‌ی یک شخص با دست‌های پایین و بالا هستند. سپس این دو کاراکتر به صورت **متوالی و با تأخیر** نمایش داده می‌شوند تا اثر **انیمیشن** (حرکت) ایجاد شود..


## ۳. قطعات مورد استفاده  
- برد آردوینو Uno  
- نمایشگر LCD 16x2 (معمولاً با کنترلر HD44780)  
- یک پاتنسیومتر ۱۰k اهم (برای تنظیم کنتراست LCD)  
- برد برد (Breadboard)  
- سیم‌های جامپر (jumper wires).

## ۴. نحوه‌ی ساخت مدار  
- **LCD**:  
  - پین 1 (VSS) → GND  
  - پین 2 (VDD) → 5V  
  - پین 3 (VO) → وسط پاتنسیومتر (برای تنظیم کنتراست)  
  - پین 4 (RS) → پین ۱۲ آردوینو  
  - پین 5 (RW) → GND (در حالت نوشتن)  
  - پین 6 (E) → پین ۱۱ آردوینو  
  - پین 7-10 (D0-D3) → نادیده گرفته می‌شوند (در حالت 4-bit)  
  - پین 11 (D4) → پین ۵ آردوینو  
  - پین 12 (D5) → پین ۴ آردوینو  
  - پین 13 (D6) → پین ۳ آردوینو  
  - پین 14 (D7) → پین ۲ آردوینو  
  - پین 15 (A) → 5V (با مقاومت ۲۲۰ اهم برای روشن کردن پس‌زمینه)  
  - پین 16 (K) → GND  



## ۵. کد برنامه (اسکچ):

```cpp
#include <LiquidCrystal.h>   // اضافه کردن کتابخانه LCD

// تعریف پین‌های LCD
const int rs = 12, en = 11, d4 = 5, d5 = 4, d6 = 3, d7 = 2;
LiquidCrystal lcd(rs, en, d4, d5, d6, d7);  // ایجاد شیء LCD

// تعریف کاراکترهای سفارشی
byte armsDown[8] = {
  0b00100,
  0b01010,
  0b00100,
  0b00100,
  0b01110,
  0b10101,
  0b00100,
  0b01010
};

byte armsUp[8] = {
  0b00100,
  0b01010,
  0b00100,
  0b10101,
  0b01110,
  0b00100,
  0b00100,
  0b00100
};

void setup() {
  lcd.begin(16, 2);   // شروع LCD با 16 ستون و 2 ردیف

  // ایجاد کاراکترهای سفارشی
  lcd.createChar(0, armsDown);   // کاراکتر 0 = دست‌های پایین
  lcd.createChar(1, armsUp);     // کاراکتر 1 = دست‌های بالا

  lcd.clear();                   // پاک کردن صفحه
}

void loop() {
  // حرکت از چپ به راست
  for (int i = 0; i <= 15; i++) {
    lcd.setCursor(i, 0);         // قرار دادن کورسور در ردیف اول
    lcd.write(byte(1));          // نمایش کاراکتر دست‌های بالا
    delay(500);                  // تأخیر ۵۰۰ میلی‌ثانیه
    lcd.clear();                 // پاک کردن صفحه

    lcd.setCursor(i, 1);         // قرار دادن کورسور در ردیف دوم
    lcd.write(byte(0));          // نمایش کاراکتر دست‌های پایین
    delay(500);                  // تأخیر ۵۰۰ میلی‌ثانیه
    lcd.clear();                 // پاک کردن صفحه
  }

  // حرکت از راست به چپ
  for (int i = 15; i >= 0; i--) {
    lcd.setCursor(i, 0);         // قرار دادن کورسور در ردیف اول
    lcd.write(byte(1));          // نمایش کاراکتر دست‌های بالا
    delay(500);                  // تأخیر ۵۰۰ میلی‌ثانیه
    lcd.clear();                 // پاک کردن صفحه

    lcd.setCursor(i, 1);         // قرار دادن کورسور در ردیف دوم
    lcd.write(byte(0));          // نمایش کاراکتر دست‌های پایین
    delay(500);                  // تأخیر ۵۰۰ میلی‌ثانیه
    lcd.clear();                 // پاک کردن صفحه
  }
}
```

### توضیح کد:  
- `armsDown` و `armsUp` — دو کاراکتر سفارشی هستند که نشان‌دهنده‌ی یک شخص با دست‌های پایین و بالا هستند.  
- `lcd.createChar(0, armsDown)` — کاراکتر دست‌های پایین را در موقعیت ۰ LCD ذخیره می‌کند.  
- `lcd.createChar(1, armsUp)` — کاراکتر دست‌های بالا را در موقعیت ۱ LCD ذخیره می‌کند.  
- `for (int i = 0; i <= 15; i++)` — حلقه‌ای که کاراکتر را از چپ به راست حرکت می‌دهد.  
- `lcd.setCursor(i, 0)` — قرار دادن کورسور در موقعیت مشخص.  
- `lcd.write(byte(1))` — نمایش کاراکتر دست‌های بالا.  
- `lcd.clear()` — پاک کردن صفحه بعد از هر نمایش برای ایجاد اثر حرکت.  
- `delay(500)` — برای تعیین سرعت حرکت.

.

## ۷. نتایج  
انیمیشن نشان داده شده در LCD برابر است با یک ادمک که بالای صفحه دستانش بالا است و به پایین صفحه میرسد, دستانش پایین میرود. این روند تا اخر صفحه ادامه پیدا میکند و سپس این ادمک مسیر برعکس میرود
.
